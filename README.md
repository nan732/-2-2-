# 南开大学高级语言程序设计 2-2 大作业

## 项目名称

**陷阱小球**

这是我这次高级语言程序设计 2-2 大作业做的一款小游戏。游戏整体是一个横版闯关加陷阱对抗的玩法，大概可以理解成：一个玩家控制小球往终点跑，另一个玩家负责在路上触发各种陷阱，阻止小球顺利通过。

虽然它看起来是一个比较简单的小游戏，但是里面其实包含了移动控制、碰撞检测、陷阱状态、游戏界面、音乐播放、胜负判断这些内容，所以做下来之后感觉还是比较完整的。

## 一开始为什么想做这个游戏

这款游戏的最初设计想法来自于我以前在 4399 上玩过的一款小游戏。那个游戏我印象挺深的，但现在再去找已经找不到了。所以这次做大作业的时候，我就想，不如凭着自己的印象和理解，把类似的玩法重新做一遍。

当然，我做的这个版本并不是完全复刻原来的游戏，因为原来的很多细节我也记不清了。所以我更多是保留了“一个人走、另一个人放陷阱”这种核心感觉，然后在这个基础上自己设计了一些机关和表现效果。

我觉得这个玩法比较有意思的地方在于，它不是单纯看谁手速快。控制小球的人要判断什么时候往前冲、什么时候停一下；控制陷阱的人也要判断什么时候按下陷阱最合适。如果按早了，小球可能还没到；如果按晚了，小球可能已经跑过去了。所以这个游戏里面其实有一点心理博弈的感觉。

## 游戏怎么玩

游戏开始后，小球会从左边出发，目标是一路走到右边的终点。

小球玩家可以用 `A / D` 或者方向键控制左右移动。这里我没有设计跳跃功能，因为我希望游戏重点放在移动节奏和陷阱触发时机上，而不是变成普通的跳跃闯关游戏。

另一个玩家可以按 `P` 键触发陷阱。游戏会自动选择离小球最近、并且还没有被使用过的陷阱来触发。这样操作会比较简单，不需要玩家去记每个陷阱对应哪个按键，也更适合展示。

游戏中的基本操作是：

- `A / D` 或方向键：控制小球左右移动
- `P`：触发最近的陷阱
- `R`：重新开始
- `ESC`：暂停游戏
- 回车或空格：开始游戏

如果小球碰到陷阱，就会失败；如果成功跑到终点，就会胜利。

## 我做了哪些陷阱

目前游戏里一共有五种陷阱。

第一种是大锤陷阱。它触发之后会有一个挥动过程，如果小球刚好在它的攻击范围里面，就会被打中。这个陷阱比较适合用来卡小球前进的节奏。

第二种是熔炉陷阱。它触发之后会在固定位置产生危险区域，小球经过的时候如果没躲开就会失败。

第三种是激光陷阱。这个陷阱的视觉效果比较明显，而且触发后比较有压迫感，适合放在中间位置打断小球玩家的节奏。

第四种是推门陷阱。它不是单纯从上往下砸，而是会对小球的位置产生影响，让小球玩家在经过的时候需要更小心。

第五种是跷跷板石块陷阱。这个陷阱我觉得比较有意思，因为它不像普通陷阱那样只是一个固定攻击范围，而是带了一点场景互动的感觉，让后半段关卡不会显得太单调。

这些陷阱都有一个共同点，就是不是按下之后立刻完全结束，而是有自己的状态变化，比如准备、触发、生效、冷却、用完。这样做的好处是陷阱看起来不会太死板，也让玩家能看到一个比较完整的过程。

## 游戏界面和效果

游戏里做了开始菜单、游戏中的提示栏、暂停界面、胜利界面和失败界面。

提示栏上会显示当前时间、小球速度、下一个最近的陷阱，还有操作提示。这样玩家在玩的时候能比较清楚地知道当前状态。

失败的时候，小球不会只是直接消失，而是会生成一些碎片效果。这个效果虽然不是特别复杂，但我觉得能让失败反馈更明显一点，看起来也比单纯切换到失败界面要自然一些。

另外游戏里还加入了背景音乐，音乐文件放在项目文件夹里，程序启动时会自动尝试加载。虽然这不是核心功能，但有音乐之后游戏展示起来会完整一些。

## 代码是怎么组织的

这个项目主要使用了这门课学习的程序设计语言，同时借助了一个图形库来处理窗口、绘图、键盘输入和音乐播放，另外也使用了构建工具来管理整个项目。

我把代码分成了几个模块：

游戏主流程模块负责整个游戏的运行，比如初始化窗口、更新游戏状态、绘制画面、判断胜利失败、重开游戏等等。

小球模块负责小球本身，包括小球的位置、速度、移动、惯性、重力和绘制。

关卡模块负责关卡内容，比如地面、终点、背景、陷阱列表，还有判断小球是否碰到陷阱。

陷阱模块是我觉得比较重要的一部分。后面的锤子、熔炉、激光、推门和石块，都是在统一的陷阱基础上继续扩展出来的。这样做的好处是，每个陷阱都有差不多的使用方式，但是具体表现又可以不一样。

界面模块负责各种界面文字，比如菜单、暂停、胜利、失败和游戏过程中的提示信息。

工具模块里面放了一些比较通用的小函数，比如限制数值范围、平滑移动、计算距离之类的。

我觉得这个项目里比较能体现面向对象思想的地方就是陷阱系统。因为不同陷阱虽然表现不同，但是它们都有类似的生命周期，所以用一个统一的基础结构来管理会比较方便。后面如果还想加新陷阱，也可以继续照着这个方式扩展，而不需要把所有逻辑都塞进一个很大的函数里面。

## 关于工具辅助

这次大作业里面也有一些地方我是借助工具一起完成的。比如游戏的启动器，还有陷阱触发逻辑、碰撞判断这两个我觉得比较难处理的部分，是在 ChatGPT 的帮助下解决和调整的。因为这些地方如果完全自己一点点试，会比较容易出错，而且调起来也比较花时间。

另外，游戏里附带的背景音乐是用 Suno 做的。音乐不是游戏的核心玩法，但是加进去之后，整体展示的时候会更有氛围一点，也不会显得画面和声音是完全分开的。

## 怎么运行

进入对应版本文件夹后，可以直接运行里面的启动脚本。

如果是苹果电脑，就运行苹果电脑对应的启动脚本。

如果是其他电脑，就运行另一个启动脚本。

如果启动脚本不能直接运行，也可以用项目里的构建配置重新构建后再运行。

## 视频讲解可以怎么说

如果录制五分钟以上的讲解视频，我大概会按照下面这个顺序讲。

首先介绍一下项目来源。我会说这个游戏的想法来自小时候或者以前在 4399 上玩过的一款小游戏，因为现在已经找不到了，所以这次就想把这种玩法重新做出来。

然后介绍游戏玩法。这里可以直接打开游戏，从菜单界面开始讲，说明小球玩家怎么移动，陷阱玩家怎么按 `P` 键触发机关，游戏目标是什么，失败和胜利分别是什么情况。

接着可以实际演示一遍。比如先让小球往前走，然后在不同位置按 `P`，展示几个陷阱的效果。这里可以重点讲一下每个陷阱不是随便加的，而是为了让关卡过程更有变化。

之后可以讲代码结构。这里不用讲得特别深，可以简单说这个项目不是所有代码都写在一个文件里，而是分成了游戏主流程、小球、关卡、陷阱、界面这些部分。这样代码会更清楚，也更符合这门课里学到的类和对象的思想。

最后讲一下自己的收获。比如通过这个项目，我练习了类的设计、继承、多态、游戏循环、碰撞检测、状态切换、项目构建，还有图形界面和音乐播放。虽然游戏规模不算特别大，但是从零开始做出一个能运行、能演示、能交互的小游戏，还是让我对项目开发有了更具体的理解。

## 总结

总体来说，这个大作业对我来说不只是写几个函数，而是把很多部分组合成一个完整程序。做的过程中也遇到了一些问题，比如陷阱触发时机怎么设计、小球碰撞范围怎么判断、游戏状态怎么切换、画面怎么画得更清楚等等。

最后完成的版本虽然还有可以继续改进的地方，比如可以加更多关卡、更多陷阱、计分系统、双人模式界面等等，但作为一次课程大作业，我觉得它已经比较完整地展示了面向对象编程和简单游戏开发的过程。
