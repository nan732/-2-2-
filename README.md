# 南开大学高级语言程序设计 2-2 大作业

##   项目名称

**陷阱小方块**

这是我这次高级语言程序设计 2-2 大作业做的一款小游戏。游戏整体是一个横版闯关加陷阱对抗的玩法，大概可以理解成：一个玩家控制小方块往终点跑，另一个玩家负责在路上触发各种陷阱，阻止小方块顺利通过。

虽然它看起来是一个比较简单的小游戏，但是里面其实包含了移动控制、碰撞检测、陷阱状态、游戏界面、音乐播放、胜负判断这些内容，所以做下来之后感觉还是比较完整的。

## 一开始为什么想做这个游戏与设计时的一些想法

这款游戏的最初设计想法来自于我以前在 4399 上玩过的一款小游戏。那个游戏我印象挺深的，但现在再去找已经找不到了。所以这次做大作业的时候，我就想，不如凭着自己的印象和理解，把类似的玩法重新做一遍。

当然，我做的这个版本并不是完全复刻原来的游戏，因为原来的很多细节我也记不清了。所以我更多是保留了一个人走、另一个人放陷阱这种核心感觉，然后在这个基础上自己设计了一些机关和表现效果。

我觉得这个玩法比较有意思的地方在于，它不是单纯看谁手速快。控制小方块的人要判断什么时候往前冲、什么时候停一下；控制陷阱的人也要判断什么时候按下陷阱最合适。如果按早了，小方块可能可以退回去；如果按晚了，小方块可能已经跑过去了。所以这个游戏里面其实有一点心理博弈的感觉。这也是我想强调的。另外，其实每个陷阱的难度和范围设计很难控制。这就导致控制方块的玩家占据了相当的优势。为了减小反复实验，调整范围的难度。我赋予了方块一定的惯性以做平衡。

## 游戏怎么玩

游戏开始后，方块会从左边出发，目标是一路走到右边的终点。

小球玩家可以用 `A / D` 或者方向键控制左右移动。这里我没有设计跳跃功能，因为我希望游戏重点放在移动节奏和陷阱触发时机上，而不是变成普通的跳跃闯关游戏。我仍然记得以前和同学在一台电脑前博弈的快乐。所以这个游戏强调两人对抗，而不是一人独秀。

另一个玩家可以按 `P` 键触发陷阱。游戏会自动选择离方块最近、并且还没有被使用过的陷阱来触发。这样操作会比较简单，不需要玩家去记每个陷阱对应哪个按键，也更适合展示。

游戏中的基本操作是：

- `A / D` 或方向键：控制方块左右移动
- `P`：触发最近的陷阱
- `R`：重新开始
- `ESC`：暂停游戏
- 回车或空格：开始游戏

如果方块碰到陷阱，就是玩家2赢；如果成功跑到终点，就是玩家1赢。

## 我做了哪些陷阱

游戏里一共有五种陷阱。

第一种是大锤陷阱。它触发之后会有一个挥动过程，如果方块刚好在它的攻击范围里面，就会被打中。但是，这个大锤我故意把它的范围做得很大来坑一下大家。实际范围是远超提示的。也许这样的设计是来自童年的恶趣味吧。

第二种是熔炉陷阱。它触发之后会落下两面墙在固定位置产生危险区域，方块被关在里面的时候，就算没躲开就会爆炸。

第三种是激光陷阱。这个陷阱的视觉效果比较明显，而且给了玩家二更多容错空间，也有更多博弈空间。不过最后激光速度我还是没有做得很快，不然就太难了。

第四种是推门陷阱。其实就是提示更少的熔炉陷阱，简单的博弈。

第五种是跷跷板加石块陷阱。我记得4399上那个游戏有类似滑坡的东西，而我做第五个陷阱时完全想不到做成什么样。所以最后我就做了一个跷跷板加石块。因为它不像普通陷阱那样只是一个固定攻击范围，带了一点场景互动的感觉，尽可能让游戏不那么单调。

这些陷阱都有一个共同点，就是都不是按下之后立刻完全结束，而是有自己的状态变化，比如蓄力、下落、发射、警告、下砸。我觉得这样会流畅一点。

## 游戏界面和效果

游戏里做了开始菜单、游戏中的提示栏、暂停界面、胜利界面和失败界面。

提示栏上会显示当前时间、小球速度、下一个最近的陷阱，还有操作提示。这样玩家在玩的时候能比较清楚地知道当前状态。

失败的时候，小球不会只是直接消失，而是会生成一些碎片效果。这个效果虽然不是特别复杂，但我觉得能让失败反馈更明显一点，看起来也比单纯切换到失败界面要自然一些。

另外游戏里还加入了背景音乐，音乐文件放在项目文件夹里，程序启动时会自动尝试加载。虽然这不是核心功能，但有音乐之后游戏展示起来会完整一些。

游戏的美术相当难做，因为我并不想从网上找素材，最后都是用代码做出来的。这一度影响了我写各种判定的难度。

## 代码是怎么组织的

这个项目主要使用了这门课学习的c++，还有一些终端代码。同时用了raylib来处理窗口、绘图、键盘输入和音乐播放，另外也使用了CMake来管理整个项目。

我把代码分成了几个模块：

游戏主流程模块负责整个游戏的运行，比如初始化窗口、更新游戏状态、绘制画面、判断胜利失败、重开游戏等等。

方块模块负责方块本身，包括方块的位置、速度、移动、惯性、重力和绘制。

关卡模块负责关卡内容，比如地面、终点、背景、陷阱列表，还有判断方块是否碰到陷阱。

陷阱模块是我觉得比较重要的一部分。后面的锤子、熔炉、激光、推门和石块，都是在统一的陷阱基础上继续扩展出来的。这样做的好处是，每个陷阱都有差不多的使用方式，但是具体表现又可以不一样。

界面模块负责各种界面文字，比如菜单、暂停、胜利、失败和游戏过程中的提示信息。

工具模块里面放了一些比较通用的小函数，比如限制数值范围、平滑移动、计算距离之类的。

## 主要文件说明

为了让文件看起来更直观，我把 3.0 版本里比较重要的代码文件改成了中文名字。大概对应关系如下：

- `主程序.cpp`：程序入口，主要负责创建游戏对象并开始运行。
- `游戏控制.h` 和 `游戏控制.cpp`：控制整个游戏流程，比如开始、暂停、重开、胜利、失败、音乐加载和画面绘制。
- `玩家方块.h` 和 `玩家方块.cpp`：负责玩家方块的移动、速度、惯性、重力、位置限制和绘制。
- `关卡.h` 和 `关卡.cpp`：负责关卡里的地面、背景、终点、陷阱列表，以及判断方块是否到达终点或碰到陷阱。
- `陷阱.h` 和 `陷阱.cpp`：负责所有陷阱的基础逻辑，以及大锤、熔炉、激光、推门、跷跷板石块这几个具体陷阱。
- `界面.h` 和 `界面.cpp`：负责菜单、提示栏、暂停界面、胜利界面和失败界面的文字显示。
- `数学工具.h` 和 `数学工具.cpp`：放一些经常用到的小工具，比如限制数值范围、平滑移动、计算距离。
- `类型定义.h`：放游戏状态、陷阱状态、碎片效果这些比较基础的数据定义。
- `CMakeLists.txt`：项目的构建配置，告诉构建工具要编译哪些文件、使用什么标准、链接什么图形库。
- `启动游戏.command`：苹果电脑上用的启动器，会自动构建并运行游戏。
- `启动游戏.bat`：其他电脑上用的启动器，也会自动构建并运行游戏。
- `177840180060595.mp3`：游戏里的背景音乐文件。


## 关于工具辅助

这次大作业里面也有一些地方我是借助工具一起完成的。比如游戏的启动器，这个实在没有了解过。还有陷阱触发逻辑、碰撞判断以及部分图像生成这几个我觉得比较难处理的部分，其实是用了GPT的，还顺便纠错了一下其它代码，升级了一下。
另外，游戏里附带的背景音乐是用 Suno 做的。音乐不是游戏的核心玩法，但是加进去之后，整体展示的时候会更有氛围一点，游玩体验好一点。

## 总结

总体来说，这个大作业对我来说还是很有趣的啊，无论是设计还是编程都挺有意思。做的过程中也遇到了很多问题，比如陷阱触发时机怎么设计、小球碰撞范围怎么判断、游戏状态怎么切换、画面怎么画得更清楚等等。不过有强大的AI帮我兜底啊。

最后完成的版本虽然还有可以继续改进的地方，比如可以加更多关卡、更多陷阱、计分系统、AI对手、双人模式界面等等，但作为一次课程大作业，我觉得它已经比较完整地展示了面向对象编程和简单游戏开发的过程，所以就只到了这一步。
